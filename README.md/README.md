## UNIX Comparison table
# UNIX Shell Comparison

| Shell | Origin & Release | Key Features | Strengths | Weaknesses | Typical Use |
|-------|-----------------|--------------|-----------|------------|-------------|
| **Bourne Shell (sh)** | AT&T Bell Labs, 1979 | Variables, control structures, built-in operators | Simple, portable, widely adopted | Limited interactivity | Legacy scripts, foundational shell |
| **C Shell (csh)** | UC Berkeley, late 1970s | C-like syntax, job control, command history | Intuitive for C programmers, interactive | Scripting quirks, less reliable for automation | Interactive use, early BSD systems |
| **Korn Shell (ksh)** | David Korn, early 1980s | Associative arrays, advanced string handling | Combines sh simplicity with csh features | Less popular today vs. bash/zsh | Commercial UNIX systems, balanced scripting |
| **Bourne Again Shell (bash)** | GNU Project, 1989 | Combines sh, ksh, csh features | Default on Linux/macOS, versatile, user-friendly | Can be slower than lighter shells (e.g., dash) | Linux scripting, interactive use, cross-platform |
| **Tcsh** | Late 1980s | Command-line editing, autocompletion | Improved csh experience, interactive enhancements | Niche adoption, overshadowed by bash/zsh | Users preferring csh lineage |
| **Z Shell (zsh)** | 1990 | Customizable prompts, spelling correction, shared history | Highly customizable, modern, active community | Learning curve for customization | Default on macOS Catalina+, developer favorite |
